The current state of our databases

üí° Explanation:

In this database infrastructure, we have several components:

  1. Authentication represents user authentication and includes fields such as email, password, UID, and display name.

  2. Doctors and Patients are collections in Firestore, where each document is identified by the respective user‚Äôs UID from the Authentication component. For Doctors, additional fields include speciality and location. For Patients, there is an age field.

  3. Used-Slots is another collection in Firestore used for storing appointment information. Each document here represents a booked time slot. It includes fields for slot time, patient UID, and doctor UID. The patient UID and doctor UID fields are foreign keys linking to the respective users‚Äô UIDs in the Auth component.

This structure allows you to associate user authentication data with specific user roles (Doctors or Patients) and manage appointment slots efficiently in the Used-Slots collection. The foreign keys in the Used-Slots collection establish relationships between patients, doctors, and their respective appointments.
